## FedRAMP Template Descriptions
### Authorization and Assessment Preparation

## CSP Annual Assessment Controls Selection Worksheet
Used to determine which FedRAMP controls must be assessed during the annual assessment. It helps CSPs and assessors identify applicable controls based on system changes, risk posture, and continuous monitoring requirements.


## FedRAMP Security Controls Baseline
Defines the security control requirements for FedRAMP systems based on impact level (Low, Moderate, High, LI-SaaS). These controls originate from the NIST SP 800-53 framework and form the foundation of FedRAMP compliance.
### FedRAMP Initial Authorization Package Checklist
A checklist used to ensure that all required documentation for a FedRAMP authorization package has been completed and submitted. It helps CSPs prepare a complete authorization package for review.

## System Security Documentation
FedRAMP High / Moderate / Low / LI-SaaS Baseline System Security Plan (SSP)
The System Security Plan (SSP) describes:
-The cloud system architecture
-Security controls implemented
-Control implementation details
-Roles and responsibilities
-Data flow and boundary definitions
The SSP is the core security document required for FedRAMP authorization.

### Readiness Assessment
FedRAMP High Readiness Assessment Report (RAR) Template
FedRAMP Moderate Readiness Assessment Report (RAR) Template
These reports are prepared by a Third Party Assessment Organization (3PAO) to evaluate whether a Cloud Service Provider is ready to pursue FedRAMP authorization.
The RAR identifies:
-System architecture readiness
-Security control implementation maturity
-Major risks and gaps
-Authorization readiness status

## Security Assessment Documentation
### FedRAMP Security Assessment Plan (SAP) Template
The SAP defines how the security assessment will be conducted by the 3PAO. It includes:
-Testing methodology
-Assessment procedures
-Scope of testing
-Control testing strategy
### FedRAMP Security Assessment Report (SAR) Template
The SAR documents the results of the security assessment conducted by the 3PAO. It includes:
-Control testing results
-Identified vulnerabilities
-Risk analysis
-Recommendations for remediation

## Assessment Supporting Documentation
### FedRAMP SAR Appendix B – Security Requirements Traceability Matrix (SRTM)
Available for Low, Moderate, and High baselines.
The SRTM maps:
-FedRAMP security controls
-Assessment procedures
-Test results
-Implementation evidence
This matrix ensures traceability between controls, testing, and findings.

### SAR Appendix A – FedRAMP Risk Exposure Table (RET)
Used to summarize identified risks, vulnerabilities, and potential impacts discovered during the assessment. It helps prioritize remediation activities.

## Continuous Monitoring
### FedRAMP Continuous Monitoring Deliverables Template
Defines required deliverables for FedRAMP continuous monitoring, such as vulnerability scans, POA&M updates, and configuration management reports.
### FedRAMP Continuous Monitoring Monthly Executive Summary Template
Provides a high-level summary of system security posture each month, including:
-Vulnerability metrics
-Incident updates
-Risk posture
-Remediation status

## Risk and Vulnerability Management
### FedRAMP POA&M Template (Plan of Action and Milestones)
Tracks identified vulnerabilities, remediation actions, responsible parties, and remediation timelines. It is one of the most critical documents used during both authorization and continuous monitoring.
### FedRAMP Vulnerability Deviation Request Form
Used when a vulnerability cannot be remediated within the required FedRAMP timeframe. The CSP submits justification and compensating controls for approval.

## Authorization Documentation
### FedRAMP ATO Letter Template
A template used by the Authorizing Official (AO) to formally grant an Authority to Operate (ATO) for a cloud system after successful security assessment and risk review.

## SSP Supporting Appendices
### SSP Appendix A – FedRAMP Security Controls
Available for:
-High baseline
-Moderate baseline
-Low baseline
-LI-SaaS baseline
These appendices list the complete set of security controls applicable to each impact level.

## SSP Appendix F – Rules of Behavior (RoB)
Defines acceptable user behavior and security responsibilities for individuals accessing the information system.

## SSP Appendix G – Information System Contingency Plan (ISCP)
Documents the procedures required to recover system operations during disruptions, including disaster recovery strategies and recovery timelines.

## SSP Appendix J – CSO, CIS, and CRM Workbook
Used to document:
-Cloud Service Offerings (CSO)
-Customer Responsibilities (CRM)
-Inherited controls and shared responsibilities
This helps clarify which security controls are managed by the CSP versus the customer.

## SSP Appendix M – Integrated Inventory Workbook
Maintains a comprehensive inventory of:
-Hardware assets
-Software assets
-Virtual resources
-System components
Required for maintaining configuration and asset management.

## SSP Appendix Q – Cryptographic Modules Table
Documents cryptographic modules used within the system, including:
-Encryption standards
-Key management methods
-FIPS 140-2 / FIPS 140-3 validation status

Purpose of This Repository
This repository is intended to:
-Provide reference documentation for FedRAMP compliance
-Help security professionals, auditors, and students understand FedRAMP documentation
-And the MOST IMPORTANT purpose it serves as a GRC learning resource for cloud security governance
